# Security Policy

> [!CAUTION]
> Do not report security vulnerabilities in issues or any other public medium.

## Reporting a Vulnerability

If you find a security vulnerability, please report it to [presently@ianhorler.dev](mailto:presently@ianhorler.dev?subject=%5BSecurity%20vulnerability%5D%20Briefly%20describe%20the%20vulnerability&body=Hi%20Presently%20Team%2C%0A%0AI%20found%20a%20security%20vulnerability%20in%20the%20Presently%20software.%20(Insert%20any%20other%20quick%20details%20here)%0A%0AThe%20issue%3A%0A(Long%20description%20of%20the%20vulnerability)%0A%0ASteps%20to%20reproduce%3A%0A(Detailed%20steps%20to%20reproduce%20the%20vulnerability)%0A%0AProblems%20caused%20by%20this%20vulnerability%3A%0A(The%20reason%20why%20this%20needs%20to%20be%20fixed)%0A%0AThanks%2C%0A(Your%20name)).
Please note that you may not receive a reply. You may not have any confirmation of it being worked on
until the public release.

In your email, please include:

 - A brief description in the subject
 - A few extra quick details
 - A long description of the vulnerability
 - Steps to reproduce it
 - Why it's a problem

To make it easier, you may use this template (prefilled if you click the email address link above):

```
To: presently@ianhorler.dev
Subject: [Security vulnerability] Briefly describe the vulnerability

Hi Presently Team,

I found a security vulnerability in the Presently software. (Insert any other quick details here)

The issue:
(Long description of the vulnerability)

Steps to reproduce:
(Detailed steps to reproduce the vulnerability)

Problems caused by this vulnerability:
(The reason why this needs to be fixed)

Thanks,
(Your name)
```
